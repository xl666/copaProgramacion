Ayuda

Generalidades entrada/salida

La revisión de los programas se realiza de forma automática, por lo que es necesario apegarse a ciertas normas de entrada y salida.

Las entradas son dadas a los programas, por lo que el programa sólo debe preocuparse por leer dicha entrada. Las entradas se dan a través de la entrada estándar (stdin). Por su parte, las salidas que genera el programa deben imprimirse en la salida estándar (stdout).

Para leer de la entrada estándar:

  • Python: var = input()
  • C: tipo var; scanf("%tipo\n", &var);
  • C++: tipo var; cin >> var;
  • Java: Scanner scan = new Scanner(System.in); String s = scan.next(); int i = scan.nextInt(); etc..

Para inprimir en la salida estándar (con un salto de línea):

  • Python: print('%s %s' % (var1, var2))
  • C: printf("%tipo %tipo\n", var1, var2);
  • C++: cout<< var1 << " " << var2 << "\n";
  • Java: System.out.println("" + var1 + " " + var2);
NOTA: Para python se utiliza python3 en el servidor, por lo que print se utiliza como una función cualquiera

Entrada

En cada problema se especifica (o se sobre entiende) la forma en que deben hacerse las lecturas. Por ejemplo, si en la definición del problema dice algo así (para llenar una matriz):
Entrada
En la primera línea se te dará el número de filas y columnas (F, C) separadas por un espacio. Las siguientes F líneas contendrán C números separados por un espacio (la especificación del espacio en realidad sirve para saber que hay varias lecturas)

. Para cada lenguaje se hará de la siguiente forma:

  • python:
    filas = int(input())
    columnas = int(input())
    matriz = [[0 for x in range(columnas)] for x in range(filas)]
    for i in range(filas):
        for j in range(columnas):
           matriz[i][j] = int(input())

    NOTA: observar las conversiones de tipo, por defecto input() regresa una cadena.
  • C:
    int f, c;
    int mat[MAX][MAX];
    scanf("%d\n%d\n",&f,&c);
    for (int i=1; i<=f; i++)
    {
        for (int j=1; j<=c; j++)
        {
           scanf("%d\n",&mat[i][j]);
        }
    }
    NOTA: observar el uso del carácter de salto de línea, de preferencia debe ponerse después de haber leído algo
  • C++ :
    int f, c;
    int mat[MAX][MAX];
    cin >> f;
    cin >> c;
    for (int i=1; i<=f; i++)
    {
        for (int j=1; j<=c; j++)
        {
           cin >> mat[i][j];
        }
    }
  • Java:
    Scanner scan = new Scanner(System.in);
    int f = scan.nextInt();
    int c = scan.nextInt();
    int[][] mat = new int[f][c];
    for(int i = 0; i < f; i++)
    {
        for(int j = 0; j < c; j++)
        {
           mat[i][j] = scan.nextInt();
        }
    }

Salida

Para la salida que debe imprimir el programa, basta con apegarse al formato específico que menciona el problema, esto incluye la impresión de espacios y saltos de línea tal cual se menciona (el ejemplo de salida de cada problema es muy útil para esto). Es importante también no imprimir nada de más como por ejemplo una cadena para pedir que se ingrese un valor

Subida del programa

Sólo se debe subir un archivo, dicho archivo es el código fuente de su programa ó, para el lenguaje de programación java, un archivo .zip(todos los archivos deben estar en el mismo package o no debe haber ningún package). Es muy importante que dependiendo del lenguaje de programación se coloque la extensión adecuada (.py, .c, .cpp, .java, .prolog, .lisp, .zip). Este archivo será compilado en el servidor. Si el código subido tiene algún error de compilación éste se reportará. Así mismo es importante que el nombre de su archivo NO contenga espacios. De preferencia abstengance de subir un archivo a menos que se esté seguro de que compila bien y de que resuelve al menos el caso de ejemplo.

Evaluación

Su programa será evaluado con los casos de pruba definidos por el académico. Dependiendo del puntaje asígnado al ejercicio y el número de casos de prueba de resuelva, será el puntaje obtenido. Por ejemplo: Si un académico definió un ejercicio con 5 casos de prueba y un puntaje de 100, cada caso de prueba valdrá 20, es decir, si se obtiene un puntaje de 80 es porque su respuesta habrá resuelto 4 de los 5 casos. Al evaluarse un programa pueden suceder 4 cosas diferentes:

  • El programa soluciona todos los casos: obtiene todo el puntaje asignado al ejercicio
  • El programa soluciona algunos casos: obtiene el puntaje proporcional al número de casos resueltos
  • El programa no soluciona ningún caso: no obtiene ningún punto
  • El programa produce un error de runtime, compilación o tiempo excedido: se le notifica el error y no obtiene ningún punto

Con la finalidad de mantener su mayor puntaje obtenido, el sistema sólo reemplazará sus respuestas cuando obtenga un puntaje más alto.
